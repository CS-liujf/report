# 在线生成听报告记录

[![GitHub Pages](https://img.shields.io/badge/GitHub%20Pages-Deployed-green.svg)](https://cs-liujf.github.io/report/)

## 简介

你是否已厌倦在研究生系统里手动录入听报告记录？有没有自动化的工具？有的，兄弟有的😎。

本项目是一个在线工具，用于自动化快速生成听报告记录的网络请求代码。通过上传包含学术报告信息的EML格式邮件，工具会解析邮件内容，然后生成可直接在研究生系统中执行的代码，简化听报告记录的登记流程。

本项目100%开源，**纯前端操作不上传任何数据**，十分安全🛡️。

## 如何使用

1. 访问[在线工具](https://cs-liujf.github.io/report/)
2. 在输入框中填写学号
3. 上传包含学术报告信息的EML格式邮件文件（可上传多个）

   **注意**，当前仅支持带有日历附件（.ics结尾的文件）的邮件。

   如何获取EML格式邮件文件呢？登录校园邮箱，点击位于上方的菜单栏“更多”。

4. 选择"仅存草稿"或"提交导师审核"

   如果选择前者，第9步之后，你还需在系统里一个个手动点击提交；若选择后者，第9步之后，本次填写的记录全部提交给导师待审核。

5. 点击"生成"按钮，系统会自动解析邮件并生成对应的JavaScript代码
6. 点击"复制"按钮将代码复制到剪贴板
7. 进入研究生系统的"听报告登记"页面，按F12打开开发者工具
8. 在控制台粘贴复制的代码并按Enter键执行
9. 手动刷新页面查看结果
